3 ОПИС ВИКОНАНОЇ РОБОТИ
3.1 Вступ
У рамках практичного завдання було обрано мову програмування Go (Golang)
та досліджено офіційні стилістичні настанови для написання якісного коду. Go —
це компільована мова програмування, розроблена в Google у 2009 році, яка поєднує
простоту, ефективність та вбудовану підтримку паралелізму.
3.2 Основні джерела стилістичних настанов
Для дослідження було використано два основні офіційні документи:
а)

Effective Go [1] — офіційний посібник, що описує філософію мови,
ідіоматичний код та найкращі практики.

б)

Code Review Comments [2] — рекомендації з перегляду коду з
конкретними прикладами та поширеними помилками.

Ці документи є стандартом в Go-спільноті, а не просто рекомендаціями.
3.3 Форматування коду
3.3.1 gofmt — офіційний інструмент
Go має унікальну особливість — офіційний інструмент форматування
«gofmt» , який автоматично приводить код до єдиного стандарту. Це усуває всі
суперечки про стиль форматування.
Основні правила форматування:
–

Відступи: Go використовує табуляцію замість пробілів. Одна табуляція
для кожного рівня вкладеності.

–

Довжина рядків: Немає жорсткого ліміту, але рекомендовано до 80-100
символів для читабельності.

–

Стиль дужок: K&R стиль — відкриваюча дужка на тому ж рядку. У Go це
вимога синтаксису, а не просто стиль.

Приклад використання «gofmt» :
До форматування:

4
package main
import "fmt"
func main(){
fmt.Println("Hello")
fmt.Println("World")
}

Після «gofmt» :
package main
import "fmt"
func main() {
fmt.Println("Hello")
fmt.Println("World")
}

3.4 Іменування
3.4.1 Пакети (модулі)
Правила іменування пакетів у Go:
–

Коротко та зрозуміло: «http» , «json» , «time»

–

Без підкреслень: «userservice» замість «user_service»

–

Тільки малі літери: «package user» , а не «package User»

–

Однина замість множини: «user» замість «users» (за винятками)

Поганий приклад:
package UserManagementService
package user_handlers
package HTTPUtils

Гарний приклад:
package user
package handler
package httputil

3.4.2 Змінні та функції
Правила іменування:
–

«MixedCaps» або «mixedCaps» стиль

–

Експортовані (публічні) починаються з великої літери: «UserID»

–

Локальні (приватні) починаються з малої літери: «userID»

5
–

Короткі імена в малих областях видимості: «i» , «r» , «w»

Приклад:
var userID int64 // локальна змінна
type User struct {
Name string // експортоване поле
}
func (u *User) Email() string { // метод без "Get"
return u.email
}

3.5 Структура веб-застосунку
Типова структура Go веб-проєкту:
mywebapp/
├── cmd/
│
└── server/
│
└── main.go
├── internal/
│
├── handler/
│
├── service/
│
└── repository/
├── pkg/
│
└── middleware/
├── go.mod
└── go.sum

# точка входу
# HTTP обробники
# бізнес-логіка
# робота з БД
# багаторазовий код

–

«internal/» — приватний код (лише для цього проєкту)

–

«pkg/» — публічний код (можна використовувати в інших проєктах)

3.5.1 Принципи модульності
а)

Розділення за відповідальністю — кожен пакет має одну чітко визначену
функцію

б)

Ієрархія залежностей — залежності йдуть в одному напрямку: handler →
service → repository

в)

Інкапсуляція логіки — використання «internal/» для приватного коду

г)

Інтерфейси для зв’язку — визначення інтерфейсів там, де вони
використовуються

3.6 Коментарі та документація
Go використовує систему «Godoc» для автоматичної генерації документації
з коментарів.
Правила коментування:
–

Коментарі починаються з імені того, що описують

–

Пишуться повними реченнями з великої літери

–

Пояснюють «чому» та «навіщо», а не «що робить код»

Поганий приклад:
// get user
func getUser(id int) {}
// This function validates the email
func validate(e string) bool {
return true
}

Гарний приклад:
// GetUser retrieves a user by ID from the database.
func GetUser(id int64) (*User, error) {
// ...
}
// ValidateEmail checks if the email has valid format.
func ValidateEmail(email string) bool {
// ...
}

3.7 Обробка помилок
Go має явну обробку помилок, що робить код більш надійним.
Основні принципи:
–

Завжди перевіряйте помилки

–

Створюйте власні типи помилок для специфічних випадків

–

Використовуйте контекстні повідомлення

–

Не ігноруйте помилки через «_»

Приклад:
// UserNotFoundError вказує, що користувача не знайдено
type UserNotFoundError struct {
UserID int64
}

7
func (e *UserNotFoundError) Error() string {
return fmt.Sprintf("user with ID %d not found", e.UserID)
}
func GetUserHandler(w http.ResponseWriter, r *http.Request) {
user, err := service.GetUser(userID)
if err != nil {
// обробка помилок з відповідними HTTP статусами
http.Error(w, err.Error(), http.StatusInternalServerError)
return
}
json.NewEncoder(w).Encode(user)
}

3.8 Test-Driven Development (TDD)
TDD — методологія розробки, де тести пишуться до коду.
Цикл TDD:
а)

«Red» — написати тест, який не проходить

б)

«Green» — написати мінімальний код для проходження тесту

в)

«Refactor» — покращити код, зберігаючи функціональність

Переваги TDD:
–

Код покритий тестами з самого початку

–

Простіший дизайн коду

–

Впевненість у рефакторингу

–

Документація через тести

–

Менше багів у продакшені

Приклад тесту:
func TestGetUserHandler(t *testing.T) {
req := httptest.NewRequest("GET", "/user?id=123", nil)
w := httptest.NewRecorder()
GetUserHandler(w, req)
resp := w.Result()
if resp.StatusCode != http.StatusOK {
t.Errorf("expected status 200, got %d", resp.StatusCode)
}
}

3.9 Інструменти для підтримки якості коду
Go має багатий екосистемі інструментів:

8
а)

«gofmt» [3] / «goimports» — автоматичне форматування

б)

«go vet» — пошук типових помилок

в)

«staticcheck» [4] — глибока статична перевірка

г)

«go test» — запуск тестів з покриттям

3.9.1 Автоматизація через CI/CD
Приклад конфігурації GitHub Actions:
on: [push, pull_request]
jobs:
test:
runs-on: ubuntu-latest
steps:
- uses: actions/checkout@v3
- uses: actions/setup-go@v4
with:
go-version: '1.21'
- name: Format check
run: gofmt -l . && test -z $(gofmt -l .)
- name: Vet
run: go vet ./...
- name: Test
run: go test -v -cover ./...

г)

е)

надійним та передбачуваним.

ж)

Ознайомлено з методологією Test-Driven Development (TDD) та її
практичним застосуванням у Go проєктах.

и)

Досліджено екосистему інструментів для автоматизації перевірки якості
коду, включаючи статичний аналіз та інтеграцію з CI/CD системами.

Отримані знання та навички дозволяють писати чистий, підтримуваний
та ідіоматичний код на мові Go, що відповідає сучасним стандартам розробки
програмного забезпечення.
